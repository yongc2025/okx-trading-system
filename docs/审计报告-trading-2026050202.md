## 一、审计背景

本次审计聚焦两个核心问题：

1. **没有参数设置页面** — 用户无法在界面上调整交易参数
2. **没有多 Key 管理** — 用户无法管理多个 OKX API Key

同时全面复查自上次审计以来的修复进展和新增发现。

## 三、问题详细分析

### 3.1 🔴 参数设置页面缺失（新发现）

**现状：**

后端已有完整的配置体系：

- `core/settings.py` — 定义了 16 个配置项的 Schema（类型、默认值、描述）
- `api/app.py` — 有 `GET/POST /api/settings` 接口
- 数据库有 `app_settings` 表

但前端没有任何设置页面，用户无法调整：

| 配置项 | 说明 | 当前默认值 |
|---|---|---|
| `leverage_long` | 做多杠杆倍数 | 3 |
| `leverage_long_max` | 做多杠杆上限 | 3 |
| `leverage_short` | 做空杠杆倍数 | 2 |
| `leverage_short_max` | 做空杠杆上限 | 2 |
| `split_threshold` | 拆单阈值 (USDT) | 800 |
| `split_random_min` | 拆单随机浮动下限 | 0.05 |
| `split_random_max` | 拆单随机浮动上限 | 0.15 |
| `limit_to_market_sec` | 限价转市价超时 (秒) | 3 |
| `confirm_before_close` | 一键全平二次确认 | false |
| `hotkey_long` | 做多快捷键 | F1 |
| `hotkey_short` | 做空快捷键 | F2 |
| `hotkey_closeall` | 全平快捷键 | F3 |
| `hotkey_scope` | 快捷键作用域 | global |
| `use_demo` | 使用模拟盘 | false |
| `toast_enabled` | 启用通知弹窗 | true |

**问题：**

1. 前端杠杆值硬编码在 JS 状态中（`S.leverage = 3`），不从后端读取
2. `leverage_long_max` / `leverage_short_max` 硬编码在前端（`S.maxLeverLong = 3`），后端 `config.py` 也是硬编码
3. `split_threshold` 只在后端拆单逻辑中使用，但用户无法调整
4. `confirm_before_close` 已定义但从未生效
5. 快捷键配置已定义但前端硬编码了 F1/F2/F3

**需要做的：**

- 新增设置页面（可以是独立页面或弹窗面板）
- 从 `GET /api/settings` 加载配置，通过 `POST /api/settings` 保存
- 前端杠杆、档位等从后端配置读取
- 快捷键配置支持自定义

### 3.2 🔴 多 Key 管理缺失（新发现）

**现状：**

后端 `CredentialManager` 已支持多 Key：

```python
# core/credentials.py
def save(self, api_key, secret, passphrase, label="default", is_demo=False):
    # 按 label 存储，支持多个

def list_labels(self) -> list[dict]:
    # 列出所有已保存的凭证

def delete(self, label="default"):
    # 按 label 删除
```

数据库 `api_credentials` 表有 `label` 字段做区分。

但前端只有一个连接弹窗：

- 只能输入一组 Key
- 无法查看已保存的 Key 列表
- 无法切换不同 Key
- 无法删除已保存的 Key
- 无法为 Key 设置别名（如"子账号A"、"实盘主号"）
- 自动连接只取第一个 Key

**用户场景：**

一个典型交易用户可能有：
- 实盘主账号 Key
- 模拟盘 Key
- 子账号 Key（不同资金规模/策略）
- 需要快速切换测试不同账号

**需要做的：**

- 新增凭证管理面板（设置页面内或独立 tab）
- 显示已保存 Key 列表（label、模拟/实盘、创建时间）
- 支持新增/编辑/删除 Key
- 支持连接测试（调用 `/api/okx/test` 或类似接口）
- 支持一键切换当前使用的 Key
- 连接弹窗改为从已保存 Key 中选择，或手动输入

## 六、结论

| 维度 | 上次评级 | 本次评级 | 说明 |
|---|---|---|---|
| 功能完成度 | B- | **C+** | 参数设置和多 Key 管理缺失是显著的功能缺口 |
| 交易安全性 | C | **C** | P0 问题全部未修复 |
| 前端闭环 | C+ | **C** | 核心交易链路有断点 |
| 可交付性 | C | **C-** | 缺少参数配置意味着用户无法按自己的策略调整 |

**核心判断：当前 trading 模块具备基本的"看行情+下单"能力，但缺少参数配置和多 Key 管理，加上 P0 问题未修复，不适合直接用于实盘交易。**

建议优先完成「第一优先级」（参数设置 + 多 Key 管理），使用户至少能按自己的策略配置参数，再逐步修复 P0 交易链路问题。
