## 一、软件简介

OKX 交易助手是一款专为 OKX 永续合约交易设计的桌面端辅助工具，核心解决三个问题：

- **快**：下单到 OKX 服务器 < 50ms，极端行情下双通道并发平仓
- **稳**：开仓自动挂止损，加仓自动重算止损价，风控规则强制执行
- **清**：自动记录每笔交易，提供胜率、盈亏比、扛单分析等数据看板

### 主要功能

| 功能 | 说明 |
|------|------|
| 一键做多/做空 | 快捷键 F1/F2，极速下单 |
| 一键全平 | F3，限价优先 → 超时自动转市价 |
| 智能拆单 | >800U 自动拆分，避免大单暴露 |
| 自动止损 | 开仓即挂止损，加仓自动更新 |
| 加仓计算器 | 一键计算 25% 加仓量并执行 |
| 数据看板 | 胜率、盈亏比、扛单分析、止损回测 |

## 二、安装与启动

### 2.1 方式一：使用安装包（推荐）

1. 获取 `OKX交易助手` 文件夹
2. 放到任意目录（建议 `C:\OKX交易助手\`）
3. 双击 `OKX交易助手.exe`
4. 浏览器自动打开交易界面

### 2.2 方式二：从源码安装

```
步骤 1: 确保已安装 Python 3.10+
步骤 2: 双击 install.bat（自动创建虚拟环境、安装依赖）
步骤 3: 双击 start.bat（自动检查环境、启动服务）
步骤 4: 浏览器访问 http://localhost:8888
```

### 2.3 启动后你会看到

```
=============================================
  OKX 交易助手 v1.0
=============================================
  地址: http://localhost:8888
  按 Ctrl+C 停止
=============================================
```

浏览器自动弹出登录界面。

> 💡 关闭浏览器不会停止服务，可以随时重新打开 `http://localhost:8888`。
> 如需停止服务，关闭命令行窗口即可。

## 三、首次使用配置

### 3.1 设置本地密码

首次打开软件，系统会要求设置**本地启动密码**：

```
┌─────────────────────────────┐
│     ⚡ OKX 交易助手         │
│     本地登录解锁             │
│                             │
│  设置本地启动密码:           │
│  [________________]         │
│  确认密码:                  │
│  [________________]         │
│                             │
│        [ 创建 ]             │
│                             │
│  密码仅存储在本地，          │
│  用于加密你的 API Key        │
└─────────────────────────────┘
```

**注意事项：**
- 密码至少 6 位
- 此密码用于加密你的 OKX API Key，**仅保存在本地**
- **请务必记住密码**，忘记后无法恢复已保存的 API Key
- 如忘记密码，可删除 `trading/db/.auth.json` 重新设置（API Key 需重新配置）

### 3.2 获取 OKX API Key

1. 登录 [OKX 官网](https://www.okx.com)
2. 点击右上角头像 → **API**
3. 点击 **创建 API Key**
4. 填写信息：
   - API 名称：随意填写（如 `交易助手`）
   - 权限：勾选 **读取** 和 **交易**
   - ⚠️ **不要勾选「提現」**
   - IP 白名单：填写你的本机 IP（可选但推荐）
   - Passphrase：设置一个密码并牢记
5. 创建成功后，记下 **API Key**、**Secret Key**、**Passphrase**

> ⚠️ Secret Key 只显示一次，请立即保存。

### 3.3 连接 OKX

登录后会弹出连接窗口：

```
┌─────────────────────────────┐
│  🔗 连接 OKX                │
│                             │
│  API Key:    [____________] │
│  Secret Key: [____________] │
│  Passphrase: [____________] │
│                             │
│  ☐ 模拟盘                   │
│                             │
│        [ 连接 ]             │
└─────────────────────────────┘
```

- 填入上一步获取的三个值
- 如需测试，勾选「模拟盘」
- 点击「连接」

连接成功后：
- 顶部状态栏显示 **已连接**（绿色圆点）
- WebSocket 状态显示 **WS 已连接**
- 系统自动保存凭证，下次启动会自动连接

### 4.2 选择币种

1. 在左上角搜索框输入币种名称（如 `BTC`）
2. 下拉列表显示匹配的交易对
3. 点击选择（如 `BTC-USDT-SWAP`）
4. 实时价格自动显示

> 💡 常用币种会自动置顶，按使用频率排序。

### 4.3 做多 / 做空

**方法一：点击按钮**
1. 选择方向：点击 **📈 做多** 或 **📉 做空**
2. 选择仓位档位：**首仓 50%** / **加仓① 25%** / **加仓② 25%**
3. 调整杠杆倍数（+/- 按钮）
4. 确认价格框显示的当前价格
5. 点击 **一键做多** 或 **一键做空**

**方法二：快捷键**
- **F1**：一键做多
- **F2**：一键做空
- **F3**：一键全平

> 💡 快捷键全局生效，不需要软件窗口在最前面。

### 4.4 加仓

1. 确保已有该币种持仓
2. 选择 **加仓①** 或 **加仓②** 档位
3. 点击对应方向的下单按钮
4. 系统自动完成：
   - 计算加仓数量（账户本金的 25%）
   - 下单
   - 撤销旧止损单
   - 以新加权均价重新挂止损单

### 4.5 一键全平

点击 **⚠️ 一键全平** 按钮（或按 F3）：

1. 查询所有当前持仓
2. 对每个持仓发出限价平仓单
3. 等待 3 秒（可配置）
4. 未成交的自动撤单，改发市价单强平

> ⚠️ 全平通过 REST + WebSocket 双通道同时发出，确保极端行情下也能成交。

### 4.6 持仓查看

右侧信息面板实时显示：
- **持仓**：币种、方向、数量、均价、浮盈浮亏
- **日志**：所有操作记录

## 五、风控规则说明

以下规则为系统强制内置，不可绕过：

### 5.1 止损规则

| 项目 | 说明 |
|------|------|
| 止损价 | 开仓价 × 10%（固定，与杠杆无关） |
| 止损类型 | 条件单，触发后以市价执行 |
| 多单止损 | 开仓价 × 0.9 |
| 空单止损 | 开仓价 × 1.1 |
| 加仓后 | 自动以新加权均价重新计算并重挂 |

**举例：**
- BTC 开仓价 50,000 USDT
- 多单止损价 = 50,000 × 0.9 = **45,000 USDT**
- 空单止损价 = 50,000 × 1.1 = **55,000 USDT**

### 5.2 杠杆限制

| 方向 | 上限 |
|------|------|
| 做多 | 最高 3x |
| 做空 | 最高 2x |

界面上的杠杆选择器不会显示超过上限的选项。

### 5.3 仓位分配

当账户可用余额 ≥ 1500 USDT 时，强制执行：

| 操作 | 使用本金 | 累计占用 |
|------|---------|---------|
| 首仓 | 50% | 50% |
| 第一次加仓 | 25% | 75% |
| 第二次加仓 | 25% | 100% |

当余额 < 1500 USDT 时，仓位分配放开，由用户自行决定。

### 5.4 智能拆单

- 单笔订单 > 800 USDT 时自动拆分
- 每笔子订单加入 ±5%~15% 随机浮动
- 目的：避免大单在订单簿中过于显眼

## 七、常见问题

### Q: 启动后浏览器没有自动打开？

手动在浏览器地址栏输入：`http://localhost:8888`

### Q: 提示"端口被占用"？

说明 8888 端口已被其他程序使用。解决方法：
```
方式一：关闭占用 8888 端口的程序
方式二：使用其他端口启动 OKX交易助手.exe 9000
         然后访问 http://localhost:9000
```

### Q: 忘记本地密码？

1. 关闭程序
2. 删除文件：`trading/db/.auth.json`
3. 重新启动，设置新密码
4. 重新配置 OKX API Key

### Q: 连接 OKX 失败？

检查清单：
- [ ] API Key 是否正确（注意不要有多余空格）
- [ ] Secret Key 是否正确
- [ ] Passphrase 是否正确
- [ ] API Key 权限是否包含「读取」和「交易」
- [ ] 是否需要勾选「模拟盘」
- [ ] 网络是否能访问 `api.okx.com`
- [ ] API Key 的 IP 白名单是否包含你的 IP

### Q: WebSocket 频繁断开？

正常现象，系统会自动重连（最大间隔 5 秒）。如持续断开：
- 检查网络稳定性
- 检查是否使用了 VPN 或代理
- 检查 OKX 服务器状态

### Q: 下单后没有自动挂止损？

可能原因：
- API Key 权限不足（需要「交易」权限）
- 网络异常导致条件单创建失败
- 检查操作日志中的详细信息

### Q: 止损价不对？

止损价 = 开仓价 × 10%（固定），与杠杆无关。这是需求文档确认的方案 A。

### Q: 如何查看数据库？

数据库文件：`trading/db/trading.db`
可用 [DB Browser for SQLite](https://sqlitebrowser.org/) 打开查看。

### Q: 数据在哪里？

| 文件 | 位置 | 说明 |
|------|------|------|
| 数据库 | `trading/db/trading.db` | 所有交易记录、配置、日志 |
| 密码哈希 | `trading/db/.auth.json` | 本地密码（非明文） |
| 运行日志 | `trading/logs/trading.log` | 按天滚动，保留 90 天 |

## 八、安全须知

### 必做

- ✅ OKX API Key **不要开启提現权限**
- ✅ OKX API 设置 **IP 白名单**（仅允许本机 IP）
- ✅ 设置 **强本地密码**（至少 6 位，建议包含字母+数字）
- ✅ 定期检查 OKX API Key 使用情况

### 禁止

- ❌ 不要将 `trading.db` 文件分享给他人
- ❌ 不要将 `.auth.json` 文件分享给他人
- ❌ 不要在公共电脑上保存 API Key
- ❌ 不要将 API Key 的 Passphrase 告诉任何人

### 软件安全特性

- API Key 使用 **AES-256** 加密存储（军事级加密）
- 密钥从你的本地密码经 **PBKDF2** 派生（600,000 次迭代）
- 所有网络请求仅发往 `api.okx.com` / `ws.okx.com`
- **无遥测、无埋点、无数据上传**
- 日志按天滚动，90 天后自动清理
